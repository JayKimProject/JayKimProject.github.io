Harness Anomaly Detector

An AI agent that analyses claude-code-harness projects for guardrail violations, stuck tasks, and root causes — and produces a structured severity report.

📄 View on GitHub 🔗 claude-code-harness

What it detects

🚫

Guardrail violations

Flags R01–R14 rule breaches: sudo attempts, force-push blocks, reviewer-role misuse, secret-file access, and more.

📋

Blocked & stalled tasks

Reads Plans.md status markers ([!], [~]) and surfaces tasks that are stuck or escalated.

📊

Git history mismatches

Cross-references completed tasks against actual commits to catch missing evidence or suspicious authorship patterns.

🔍

Tool failure patterns

Spots repeated post_tool_failure and permission_denied events that indicate systemic rather than one-off problems.

Quick start

# Install
pip install anthropic

# Set your API key
export ANTHROPIC_API_KEY="sk-ant-..."

# Run against a harness project
python anomaly_agent.py \
  --project-root /path/to/harness-project \
  --output anomaly_report.md

Agent tools

ToolDescription
read_event_logsReads all JSONL event files from .claude/state/
read_guardrail_violationsParses events for denials, violations, and tool failures grouped by rule
read_git_logRetrieves recent commit history, branch, and diff statistics
read_plansReads Plans.md and extracts status counts and blocked tasks
read_fileReads any project file safely (path-traversal protected)
list_directoryLists files and folders at any path inside the project

Report format

Executive summary

High-level overview of findings, total anomaly count, and overall health signal.

Per-anomaly block

Description · HIGH MEDIUM LOW severity · evidence · root cause

Remediation steps

Actionable recommendations tied to specific guardrail rules, tasks, or commit references.
